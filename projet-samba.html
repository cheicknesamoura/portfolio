Contrôleur de domaine sous Linux (Samba4)

Contexte du projet

Pour démontrer l'interopérabilité des systèmes et proposer une alternative open-source aux solutions propriétaires, j'ai déployé un contrôleur de domaine Active Directory complet en utilisant un serveur Linux et Samba4. Ce projet prouve qu'il est possible de gérer un parc Windows de manière centralisée avec des outils libres.

Mes missions réalisées

  • Installation et configuration réseau de base d'un serveur Linux en ligne de commande.
  • Installation des paquets requis (Samba4, Kerberos) et configuration du service DNS.
  • Provisionnement du domaine ("Samba-tool domain provision") pour promouvoir le serveur en tant que contrôleur de domaine.
  • Intégration de postes clients Windows au domaine Linux.
  • Administration du serveur Linux à distance en utilisant les outils RSAT depuis un poste client Windows.
